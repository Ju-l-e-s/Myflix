# 📜 DEVELOPER INSTRUCTIONS (Go/MyFlix)

## 🛠 Rule 1: Strict Error Handling
All errors must be handled with clear context.
- **Mandatory Wrapping**: Use `fmt.Errorf("context: %w", err)`.
- **Logging**: Use `log/slog` for structured logs with attributes (e.g., `slog.Error("msg", "err", err, "id", id)`).

## 📦 Rule 2: Encapsulation & Structure
- Business logic **MUST** reside within `/scripts/internal`.
- Every new system dependency or API client must have its own sub-package in `/internal`.
- Never export global variables (except for immutable constants).

## 🔌 Rule 3: Interfaces & Dependency Injection
To reduce the high coupling identified during the audit:
- Define **interfaces** for third-party services (ArrClient, AI, System).
- Use these interfaces in Handler constructors.
- Example: `type MediaManager interface { Search(q string) []Media }`.

## 💾 Rule 4: Storage & Safety
- Always verify disk mounts before any write operation (`verify_data_mounts.sh`).
- Caches (e.g., posters, library) must have a defined TTL to prevent memory bloat.
- Prefer file streaming or direct links over transferring raw files via the Telegram bot.
